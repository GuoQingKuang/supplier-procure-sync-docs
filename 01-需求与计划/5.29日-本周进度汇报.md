# 本周开发进度汇报

> 汇报时间：2026-05-29

---

## 一、本周已完成功能

### 1. 供应商认证体系
- **供应商登录/登出**（`POST /auth/login`、`POST /auth/logout`）：基于用户名密码认证，支持单设备登录（新登录自动踢掉旧设备），登出时 token 加入 Redis 黑名单
- **企业微信 OAuth2 单点登录**（`GET /oauth2/login`）：管理端通过企业微信扫码登录，JWT 重定向到前端，支持踢掉旧设备
- **JWT 鉴权拦截器**：支持 `supplier` / `admin` 双角色，从 Header 或 URL 参数取 token，兼容不同鉴权需求

### 2. 管理端供应商账号管理
- **供应商账号 CRUD**：列表分页查询（关键字搜索、状态过滤）、账号创建、详情查看、编辑（密码/状态修改）、状态更新（启用/禁用）
- **金蝶供应商搜索**：对接金蝶 K3Cloud，按关键词搜索供应商主数据，辅助创建账号时选择目标供应商

### 3. 采购订单列表
- **订单分页查询**（`GET /order/list`）：查询金蝶 ERP 采购订单（SQL Server），支持订单编号模糊搜索、日期范围过滤，支持查询某条订单的剩余收料数量

### 4. 发货提交流程
- **提交发货**（`POST /delivery-notice/submit`）：校验发货数量不超过 ERP 剩余收料数量 → 写入本地送料通知单 → 调用金蝶 Push 下推收料通知单 → 调用金蝶 Save 修改收料数量 → 失败自动补偿作废本地通知单
- **通知单列表**（`GET /delivery-notice/list`）：供应商查看自己的送料通知单，成功记录关联金蝶收料单详情，失败记录展示错误信息，支持多分录合并展示
- **通知单号生成**：基于 Redis INCR 的按日递增序号，含降级方案（Redis 不可用时用时间戳）

### 5. 金蝶 API 对接
- **KingdeeApiService 封装**：Push（下推）、Save（保存）、Cancel（作废）三个核心操作，统一的 JSON 请求构建和响应解析，批量修改收料数量的 `updateReceiveQtyBatch` 方法
- **金蝶测试 Demo**：Push、Save、Cancel、View 四个独立测试脚本

### 6. 基础设施
- 多数据源配置（MySQL + SQL Server）
- Redis + Redisson 缓存和分布式锁基础
- MyBatis-Plus 分页插件 + 乐观锁插件
- Apollo 配置中心集成
- Spring Security + OAuth2 Client
- SpringDoc OpenAPI 文档（`/swagger-ui.html`）
- 全局异常处理、分页响应基类
