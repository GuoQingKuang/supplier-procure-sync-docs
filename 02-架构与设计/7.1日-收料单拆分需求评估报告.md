# 收料单拆分需求评估报告

## 一、需求背景

仓库反馈：目前供应商一次发货包含多种物料时，金蝶系统只会生成**一张收料通知单**。如果其中某个物料有问题（数量不对、质量不合格等），仓库需要对整张收料单进行操作，影响其他正常物料的收料效率。

**需求**：按物料编码拆分，相同物料编码的物料行归到同一张收料单中。例如30行物料包含20种不同物料编码，则生成20张收料单。

---

## 二、当前情况

| 项目 | 现状 |
|------|------|
| 供应商操作 | 勾选多行物料 → 生成预发货单 → 打印 → 实际送货 → 回填库位 → 正式发货 |
| 金蝶收料单 | 1张（包含所有物料行） |
| 打印 | 供应商打印1张送货单 |
| 仓库收料 | 在金蝶操作这1张收料单 |

---

## 三、推荐方案

### 核心思路

预发货单生成后，供应商可**按需点击"按物料拆分"**，将一张预发货单拆分成多张子单。子单独立管理，各自生成一张金蝶收料单。供应商可通过**合并打印**功能一次性打印所有子单，仓库在金蝶看到多张独立收料单。

### 操作流程

```
1. 供应商勾选30行物料 → 生成1张预发货单（YFH001）
         ↓
2. （可选）点击"按物料拆分" → 自动拆成20张子预发货单
         ↓
3. 勾选所有子单 → 点击"合并打印" → 生成1张送货单（30行）
         ↓
4. 供应商拿着送货单 + 实物物料 → 送到仓库
         ↓
5. 仓库告知库位信息 → 供应商回填到各子单
         ↓
6. 勾选所有子单 → 点击"批量发货" → 生成20张金蝶收料单
         ↓
7. 仓库在金蝶看到20张独立收料单，分开收料
```

### 拆分说明

- **可选操作**：供应商可自行决定是否拆分。物料种类少时可不分，种类多时拆分更方便仓库收料
- **拆分时机**：在预发货单生成后、正式发货前的任意时间
- **拆分规则**：按物料编码自动分组，相同物料编码归到同一张子单

### 失败处理

批量发货时如果20张中有2张下推失败：
- 成功的18张正常流转，仓库可以正常收料
- 失败的2张标记为"发货失败"，供应商可以单独重新操作
- 不会因为部分失败导致全部作废重做

---

## 四、方案对比

| 对比项 | 方案一：不拆分 | 方案二：手动拆分（推荐） |
|--------|---------------|------------------------|
| 供应商操作 | 一次操作 | 拆分后需批量操作（提供批量发货功能） |
| 打印 | 打印1张 | 合并打印1张 |
| 仓库体验 | 金蝶1张大收料单 | 金蝶多张独立收料单 |
| 数据一致性 | 好（1对1） | 好（子单各自1对1） |
| 失败影响 | 整体失败需重做 | 单张失败不影响其他 |
| 开发工作量 | 无 | 中等 |

---

## 五、对供应商的影响

| 方面 | 影响说明 |
|------|---------|
| 预发货操作 | **无变化**，勾选物料后生成预发货单 |
| 拆分操作 | 新增可选功能，按需点击拆分 |
| 打印 | **无变化**，合并打印仍为1张送货单 |
| 正式发货 | 拆分后使用批量发货功能，一次操作完成 |
| 失败重试 | 某张失败时，只需重试该张，不影响其他 |

---

## 六、新增功能

| 功能 | 说明 |
|------|------|
| 按物料拆分 | 预发货详情页新增按钮，点击后按物料编码自动拆分成多张子单 |
| 合并打印 | 勾选多张预发货单后一次性打印，生成包含所有物料行的送货单 |
| 批量发货 | 勾选多张草稿态预发货单后一次性发货，系统逐张处理并返回结果 |
| 批量提交表头 | 勾选多张已发货态预发货单后一次性提交审核 |
| 失败重试 | 单张发货失败后可单独重试 |
| 子单关联展示 | 列表页可查看同一母单下的子单列表 |

---

## 七、风险提示

| 风险 | 说明 | 应对 |
|------|------|------|
| 部分失败 | 批量发货时某张下推失败，其他成功 | 成功的正常流转，失败的单独重试 |
| 接口调用增加 | 拆分后调用金蝶次数增加 | 批量发货时加调用间隔，避免限流 |
| 操作复杂度 | 拆分后子单数量增多 | 提供批量操作功能，列表折叠展示 |

---

## 八、开发工作量预估

| 模块 | 工作量 |
|------|--------|
| 后端拆分逻辑 | 1-2天 |
| 数据库字段调整 | 0.5天 |
| 合并打印功能 | 1天 |
| 批量发货功能 | 1天 |
| 前端列表展示 | 1-2天 |
| 测试验证 | 2天 |
| **合计** | **约6.5-8.5天** |

---

## 九、结论

推荐采用**手动拆分方案**，供应商按需决定是否拆分，拆分后通过合并打印和批量发货功能降低操作复杂度。仓库获得独立收料单，数据一致性有保障，开发周期约1-2周。
