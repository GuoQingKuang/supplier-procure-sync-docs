## 跨系统容错机制

### 发货提交流程

```
1. 校验发货数量（查 ERP 剩余数量）
2. 写本地 delivery_notice 表（status=发货中）
3. 调金蝶 Push API 下推收料通知单
4. 调金蝶 Save API 批量修改交货数量
5. 全部成功 → status=已发货
6. 失败 → 补偿删除金蝶收料单 + status=已作废
```

### 失败场景与处理

| 场景 | 处理方式 |
|------|---------|
| Push 失败 | 标记已作废 |
| Push 成功，Save 失败 | 删除收料单 + 标记已作废 |
| Push 返回失败但金蝶已处理（假失败） | 记录日志，运维手动排查 |
| Push 成功但 Save 失败且删除也失败 | 标记异常状态 |

### 解决方案

通过 `T_PUR_RECEIVEENTRY_LK` 关联表查询映射。

```java
List<ReceiveEntryMappingVo> mappings = self.queryReceiveEntryMapping(pushResult.getReceiveBillId());
```
