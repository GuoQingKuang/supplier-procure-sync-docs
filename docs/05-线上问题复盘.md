## 问题描述

金蝶下推收料单后，采购订单分录与收料单分录不匹配，导致交货数量设置错误。

### 3. 定位根因

金蝶 Push 返回的分录 ID 顺序和前端提交的不一致，不能按顺序对应。

### 4. 解决方案

通过 `T_PUR_RECEIVEENTRY_LK` 关联表查询采购订单分录与收料单分录的对应关系。

```java
List<ReceiveEntryMappingVo> mappings = self.queryReceiveEntryMapping(pushResult.getReceiveBillId());
```
