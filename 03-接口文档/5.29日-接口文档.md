# 供应商物料系统接口文档

## 概述

本文档定义供应商物料系统（procure-sync）的后端 API 接口。

**核心原则**：订单和送料数据主要依赖金蝶 K3 Cloud 系统，本地存储账号信息和送料发货原始记录。

## 数据库表结构

### `supplier_account`（供应商账号表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| supplier_id | BIGINT | 金蝶供应商内码 (FSUPPLIERID) |
| supplier_code | VARCHAR | 金蝶供应商编码 |
| supplier_name | VARCHAR | 供应商名称 |
| username | VARCHAR | 登录用户名（唯一） |
| password | VARCHAR | 登录密码（明文） |
| status | TINYINT | 0-禁用 1-启用 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |
| create_by | BIGINT | 创建人 |

### `admin_user`（管理员用户表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| wx_user_id | VARCHAR(64) | 企业微信 userId（唯一） |
| name | VARCHAR(64) | 姓名 |
| role | VARCHAR(32) | 角色标识（如 admin） |
| status | TINYINT | 0-禁用 1-启用 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |

> 备注：后续可能需要增加字段（部门、手机号、hrmUserId 等）

### `delivery_notice`（送料通知单表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| notice_no | VARCHAR(64) | 送料通知单号 |
| supplier_id | BIGINT | 供应商内码（索引） |
| supplier_code | VARCHAR(64) | 金蝶供应商编码 |
| supplier_name | VARCHAR(256) | 供应商名称 |
| ship_time | DATETIME | 发货时间（索引） |
| status | TINYINT NOT NULL DEFAULT 1 COMMENT '状态: 1-发货中 2-已发货 3-已作废 4-异常' |
| kingdee_receive_bill_id | BIGINT | 金蝶收料单唯一标识（内码） |
| kingdee_receive_bill_no | VARCHAR(64) | 金蝶收料通知单编号（FBILLNO） |
| material_data | JSON | 原始输入信息（物料明细 JSON 数组，含分录ID、物料编码、版本、名称、规格、发货数量等） |
| kingdee_push_params | JSON | 调用金蝶下推 API 的传参数据（留样） |
| kingdee_update_params | JSON | 调用金蝶修改收料通知单 API 的传参数据（留样） |
| fail_reason | VARCHAR(512) | 失败原因 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |
| create_by | BIGINT | 创建人 |

> **说明**：供应商端列表通过 `kingdee_receive_bill_id` 关联查询金蝶收料单获取物料明细；管理端直接查本地表，点击详情查看 `material_data` 中的原始输入信息。

## 认证方式

所有接口（除登录外）需在请求 Header 中携带 JWT token：

```
Authorization: Bearer <token>
```

或通过 URL 参数传递：

```
?auth_code=<token>
```

JWT token 中包含 role claim（`supplier` / `admin`），supplier 角色携带 `supplierId`，admin 角色携带用户 ID。

## 一、认证模块

### 1.1 供应商登录

```
POST /auth/login
```

**是否需要数据库**：是，查 `supplier_account` 表

**请求体**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | String | 是 | 登录用户名 |
| password | String | 是 | 登录密码 |

**响应示例**：

```json
{
  "code": 0,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "supplierInfo": {
      "supplierId": 12345,
      "supplierCode": "SUP001",
      "supplierName": "东莞精密制造有限公司"
    }
  }
}
```

**处理流程**：
1. 验证 `supplier_account` 表中的用户名和密码（明文比对）
2. 检查账号状态（status = 1）
3. 返回 JWT token，token 中携带 supplierId

---

### 1.2 管理员 OAuth2 登录

```
GET /oauth2/login
```

**是否需要数据库**：是，查 `fscut_user_ref` 表

**请求方式**：企业微信 OAuth2 授权回调，由 Spring Security OAuth2 自动处理。用户通过企业微信授权后，回调到此接口，自动生成 token 并重定向到前端页面。

**处理流程**：
1. Spring Security OAuth2 拦截请求，完成企业微信授权
2. 用 wxUserId（given_name）查 `fscut_user_ref` 表，匹配人员信息
3. 生成 JWT token（携带 hrmUserId），重定向到前端页面，URL 中携带 token
4. 前端收到 token 后存入 localStorage
5. **单点登录控制**：新登录会将旧 token 加入黑名单（踢掉旧设备），同一账号只能在一个设备登录

> 备注：`AdminOAuth2LoginReq`、`AdminOAuth2LoginResp`、`AdminInfoResp` 等 VO 已预留，后续可能改为 POST 传参方式。

---

### 1.3 登出

```
POST /auth/logout
```

**状态**：已实现

**响应示例**：

```json
{
  "code": 0,
  "message": "登出成功"
}
```

**处理流程**：
1. 从 Header 或 URL 参数提取 token
2. 解析 token 获取剩余过期时间
3. 将 token 加入 Redis 黑名单，TTL = 剩余过期时间
4. 后续请求携带该 token 时拦截器拒绝（40100）

**说明**：客户端收到响应后清除本地存储的 token。

### 2.1 获取采购订单列表

```
GET /order/list
```

**是否需要数据库**：否，直接查金蝶 SQL Server 数据库

**请求头**：需要认证（供应商）

**查询参数**（query string）：

|------|------|------|------|

| pageSize | Integer | 否 | 页大小，默认 10 |
| billNo | String | 否 | 单据编码模糊搜索 |

| dateTo | String | 否 | 采购日期截止（yyyy-MM-dd） |


**响应示例**：

```json
{

  "data": {
    "total": 150,
    "success": true,
    "current": 1,
    "pageSize": 50,
    "data": [
      {
        "billType": "采购订单",
        "purchaseDate": "2024-03-15",
        "billNo": "POCG20240315001",
        "supplierName": "苏州颐和精密机械有限公司",
        "materialCode": "MAT-2024-001",
        "materialVersion": "V2.0",
        "materialName": "精密齿轮组件",
        "spec": "Φ80×20齿",
        "quantity": 500,
        "remainQty": 200,
        "entryId": 1001
      },
      {
        "billType": "采购订单",
        "purchaseDate": "2024-03-15",
        "billNo": "POCG20240315001",
        "supplierName": "苏州颐和精密机械有限公司",
        "materialCode": "MAT-2024-002",
        "materialVersion": "V1.5",
        "materialName": "传动轴配件",
        "spec": "L=350mm",
        "quantity": 300,
        "remainQty": 150,
        "entryId": 1002
      }
    ]
  }
}
```

**处理流程**：
1. 从 JWT token 获取 supplierId（ThreadLocal 中为 `supplierId`）
2. 按 supplierId 查金蝶 SQL Server 采购订单表，仅返回状态正常且剩余收料数量 > 0 的物料行
3. 支持单据编码、物料编码模糊搜索，以及采购日期范围过滤（参数为空时不拼接条件）
4. 按采购日期倒序、单据编码、分录内码排序
5. 返回扁平化物料行列表，分页按物料行计数，同订单连续排列（前端用底色区分订单）

**备注**：`remainQty` 为剩余可发货数量（采购数量 - 基本单位关联数量）。不使用金蝶 `FREMAINRECEIVEQTY` 字段，因为该字段需审批完成后才更新，无法实时反映已下推收料单的占用量。

### 2.2 提交发货

```
POST /delivery-notice/submit
```

**是否需要数据库**：是，写 `delivery_notice` 表

**请求头**：需要认证（供应商）

**请求体**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| items | Array | 是 | 发货物料列表（至少1条，支持跨订单） |
| items[].purchaseOrderNo | String | 是 | 采购订单编号（金蝶 FBillNo） |
| items[].entryId | Long | 是 | 采购订单分录内码（FEntryID） |
| items[].materialCode | String | 是 | 物料编码 |
| items[].materialVersion | String | 是 | 物料版本 |
| items[].materialName | String | 是 | 物料名称 |
| items[].materialSpec | String | 是 | 规格 |
| items[].deliveryQty | Integer | 是 | 本次发货数量 |

**请求体示例**：

```json
{
  "items": [
    {
      "purchaseOrderNo": "POCG20240315001",
      "entryId": 1001,
      "materialCode": "MAT-2024-001",
      "materialVersion": "V2.0",
      "materialName": "精密齿轮组件",
      "materialSpec": "Φ100×50mm",
      "deliveryQty": 100
    },
    {
      "purchaseOrderNo": "POCG20240318003",
      "entryId": 2001,
      "materialCode": "MAT-2024-001",
      "materialVersion": "V2.0",
      "materialName": "精密齿轮组件",
      "materialSpec": "Φ100×50mm",
      "deliveryQty": 200
    }
  ]
}
```

**成功响应**：

```json
{
  "code": 0,
  "data": {
    "noticeNo": "SN20240320001"
  }
}
```

**失败响应**：

```json
{
  "code": 50000,
  "message": "发货失败: 金蝶下推失败: 供应商信息不匹配",
  "data": null
}
```

**处理流程**：
1. 从 ThreadLocal 获取 supplierId，查 `supplier_account` 表获取供应商信息
2. 校验发货物料列表非空
3. 批量查金蝶 SQL Server 获取每个物料的剩余收货数量（`ErpMapper.selectBatchRemainQty`），逐条校验 `deliveryQty ≤ remainQty`（超发返回 40000 参数错误）
4. 写入本地 `delivery_notice` 表（status = 1 发货中），保存物料明细 JSON（提交前的原始数据），生成通知单号（格式：SN + yyyyMMdd + 4位序号，通过 Redis INCR 生成）
5. 将所有物料分录合并为一次金蝶下推 API 调用（支持跨订单合并），金蝶按供应商+收货方自动分组，生成**一张**收料通知单
6. 下推成功后，将金蝶返回的收料单内码和编号更新到本地记录
7. 调金蝶 Save API 批量修改收料数量为实际发货数量：
   - 若金蝶返回 1 个收料单分录（合并场景）：所有发货数量求和，设到该分录
   - 若金蝶返回 N 个收料单分录（1:1 映射）：每个分录对应一个发货物料项
8. 全部成功：更新本地 status = 2（已发货），返回送料通知单号
9. 下推失败：更新本地 status = 3（已作废），记录 fail_reason，抛出 50000 异常
10. Save 失败：调金蝶作废 API 作废收料单 → 更新本地 status = 3（已作废，作废成功）或 4（异常，作废也失败），记录 fail_reason，抛出 50000 异常

```json
            "materialCode": "MAT-2024-002",
            "materialVersion": "V1.5",
            "materialName": "传动轴配件",
            "spec": "Φ80×40mm",
            "deliveryQty": 50
          }
        ]
      }
    ],
    "warning": null
  }

```

**降级响应**（金蝶不可用时）：

```json
{
  "code": 0,
  "data": {
    "notices": {
      "total": 0,
      "success": true,
      "current": 1,
      "pageSize": 10,
      "data": []
    },
    "failedNotices": [...],
    "warning": "金蝶系统暂时不可用，成功记录暂无法查询"
  }
}
```

**处理流程**：
1. 查本地 `delivery_notice` 表（按 supplier_id 过滤），分离成功记录（有 `kingdee_receive_bill_id`）和失败记录
2. 成功记录：尝试查金蝶收料通知单表获取物料明细，合并本地状态信息（发货时间、状态、状态文本）和金蝶返回的物料明细
3. 失败记录：直接从本地 `material_data` JSON 读取物料明细
4. 成功记录按通知单维度分组后进行分页（按 shipTime 倒序），失败记录全量返回（不分页）
5. 若金蝶数据库连接超时或不可用，成功记录返回空数组，失败记录正常返回，并附加 `warning` 提示

**状态枚举**：

| 状态码 | 状态文本 | 说明 |
|--------|----------|------|
| 1 | 发货中 | 已提交，金蝶处理中 |
| 2 | 已发货 | 金蝶下推和 Save 均成功 |
| 3 | 已作废 | 金蝶下推/Save 失败，或 Save 失败后收料单已成功作废 |
| 4 | 异常 | Save 失败且收料单作废也失败 |
